# Sprint 4: Portfolio page and refactors and fixes

- **What was done:** Added logout in the header and replaced the hardcoded `isLoggedIn` with a real status derived from the token. Then built the Portfolio page: a table of assets with balances, total value, a distribution chart, tabs, pagination, a loader and error handling, and a guard so only logged-in users with a saved API key can open it. At first I loaded the prices with a normal request, but the data didn't update on its own. So I integrated a WebSocket service my colleague wrote, and now the prices update live.
- **Problems:** The user menu always showed Logout, even when I wasn't logged in. While digging into it I also noticed we have two parallel auth implementations — a signals service and an RxJS store — that can get out of sync. In general RxJs topic is very complicated to me. The problem lso was that initially I just loaded data into table and it wasn't dynamically updated because it wasn't implemented through websockets and also I didn't use signls in the table.
- **Solutions:** Turned out `isLoggedIn` was hardcoded to `true`, so I derived it from the token instead. Since the token is shared, it updates on login and logout no matter which service did it. For the two auth implementations I decided to keep both for now, because the login state is token-based anyway, and merging them into one is a separate task.
- **What I learned:** How the token ties the whole auth flow together — the guard, the interceptor and `isLoggedIn` all read the same shared token. And when it's worth refactoring duplicate services versus leaving it as tech debt for later. Learned how to load data through observables, for the first time worked with websockets and also wrote a guard myself
