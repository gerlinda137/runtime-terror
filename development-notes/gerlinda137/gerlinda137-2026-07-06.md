# Sprint 4: Portfolio page and refactors and fixes

- **What was done:** Added logout in the header and replaced the hardcoded `isLoggedIn` with a real status derived from the token.
- **Problems:** The user menu always showed Logout, even when I wasn't logged in. While digging into it I also noticed we have two parallel auth implementations — a signals service and an RxJS store — that can get out of sync.
- **Solutions:** Turned out `isLoggedIn` was hardcoded to `true`, so I derived it from the token instead. Since the token is shared, it updates on login and logout no matter which service did it. For the two auth implementations I decided to keep both for now, because the login state is token-based anyway, and merging them into one is a separate task.
- **What I learned:** How the token ties the whole auth flow together — the guard, the interceptor and `isLoggedIn` all read the same shared token. And when it's worth refactoring duplicate services versus leaving it as tech debt for later.
- **Plans:** Merge the two auth services into one (RxJS), and start on the portfolio page.
- **Time spent:** 10 hours
