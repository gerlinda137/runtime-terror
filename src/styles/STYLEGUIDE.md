# Style Guide — CSS-переменные и Material

Документ объясняет, **какие переменные у нас есть и где их использовать**. Все токены объявлены в [`app-theme.scss`](./app-theme.scss).

## Главный принцип: 3 слоя

```
Палитра (сырые цвета)  →  Семантика (роли)  →  Мост Material  →  Компоненты
   --primary-100            --surface             --mat-sys-*        используют
   --base-0                 --text-primary                           семантику и роли
```

1. **Палитра** — сырые цвета из макета. Меняются только при смене дизайна.
2. **Семантика** — «роли» (фон, поверхность, текст). Ссылаются на палитру.
3. **Мост Material** — отдаёт семантику в токены, которые читают компоненты Material.
4. **Компоненты** — используют **семантику и роли**, а НЕ сырые `--primary-100`.

> ⚠️ Правило №1: в компонентах не используем сырую палитру (`--primary-100`, `--base-20`) напрямую — берём семантику (`--surface`, `--text-secondary`) или роли Material (`--mat-sys-*`). Сырые токены — только источник для семантики.

---

## 1. Палитра (raw) — НЕ использовать в компонентах напрямую

| Группа | Токены | Назначение |
|---|---|---|
| Primary | `--primary-100 … --primary-20` | синий бренд-цвет (100 — насыщенный, 20 — самый светлый) |
| Base | `--base-100 … --base-0` | нейтральные: 100 — тёмный текст, 0 — белый |
| Success | `--success-100 … --success-20` | зелёный (рост, успех) |
| Error | `--error-100 … --error-20` | красный (ошибки, падение) |

---

## 2. Семантика — ЭТО используем в компонентах

| Токен | Что значит | Где применять |
|---|---|---|
| `--background` | подложка/canvas (светло-серый) | фон страницы под карточками |
| `--surface` | поверхность (белый) | карточки, сайдбар, хедер, панели |
| `--surface-variant` | приглушённая поверхность | ховер-фоны, разделители |
| `--card-bg` | фон карточки (= surface) | блоки `.card` |
| `--card-border` | граница карточки | бордеры панелей |
| `--text-primary` | основной текст | заголовки, основной контент |
| `--text-secondary` | вторичный текст | подписи, лейблы, иконки |
| `--icon-color` / `-hover` / `-active` | цвета иконок | интерактивные иконки |
| `--success` / `--danger` | статусные цвета | рост/падение, ошибки |

---

## 3. Роли Material (`--mat-sys-*`) — для стилизации компонентов Material

Компоненты Angular Material (`mat-button`, `mat-form-field`, `mat-nav-list`…) читают **только** `--mat-sys-*`. Мы маппим на них свою палитру в мосте.

| Роль | На что замаплено | Когда использовать |
|---|---|---|
| `--mat-sys-primary` | `--primary-100` | акцентные элементы, активные кнопки |
| `--mat-sys-on-primary` | белый | текст/иконки НА primary |
| `--mat-sys-primary-container` | `--primary-20` | светлый акцентный фон (активный пункт меню) |
| `--mat-sys-on-primary-container` | `--primary-100` | текст/иконки НА primary-container |
| `--mat-sys-surface` | `--surface` | поверхности Material |
| `--mat-sys-surface-variant` | `--surface-variant` | ховеры, второстепенные поверхности |
| `--mat-sys-on-surface` | `--text-primary` | текст на поверхности |
| `--mat-sys-outline` | `--card-border` | рамки полей/обводки |
| `--mat-sys-error` | `--danger` | состояние ошибки |

> ⚠️ Правило №2: для подсветок типа «светлый фон + цветной текст» используем **пары контейнеров** (`*-container` / `on-*-container`), а не сырые цвета. Пример — активный пункт меню: фон `--mat-sys-primary-container`, текст `--mat-sys-on-primary-container`.

## 4. Отступы и радиусы

| Токен | Значение | |
|---|---|---|
| `--padding-extra-sm` | 4px | |
| `--padding-xxs` | 8px | |
| `--padding-xs` | 12px | |
| `--padding-s` | 16px | |
| `--padding-sm` | 20px | |
| `--padding-m` | 24px | |
| `--padding-l` | 32px | |
| `--padding-card` | 20px | внутренний отступ карточки |
| `--padding-item` | 12px | строки/элементы списка |
| `--padding-item-child` | 8px | вложенные мелкие элементы |
| `--radius-lg` / `--radius-md` / `--radius-sm` | 16 / 12 / 6px | скругления |

> ⚠️ Правило №3: никаких магических `px` в стилях — только токены отступов/радиусов.

## 5. Типографика — через директиву/миксины, НЕ переменные

Размеры и начертания шрифта — это **не CSS-переменные**. Есть два способа:

- **В шаблоне** (предпочтительно): директива `appTypography`
  ```html
  <h2 appTypography="heading-3">Заголовок</h2>
  ```
- **В SCSS**: миксины из [`mixins/typography.scss`](./mixins/typography.scss)
  ```scss
  @use '../../../styles/mixins/typography.scss' as *;
  .title { @include medium-text-semibold; }
  ```

Варианты: `heading-1…6`, `display-1`, и текстовые `double-extra-large / extra-large / large / medium / small / extra-small` — каждый в `-semibold` и `-regular`.

> ⚠️ Правило №4: НЕ используем `var(--font-*)` / `var(--weight-*)` — таких токенов нет.

---

## 6. Тёмная тема

Включается атрибутом `data-theme="dark"` на `<html>`. В блоке `[data-theme='dark']` переопределяется **только палитра** (`--base-*`, `--primary-*` и т.д.) + подложка `--background`. Вся семантика и роли Material завязаны на палитру, поэтому **переворачиваются автоматически** — отдельно дублировать их в тёмной теме не нужно.

> ⚠️ Правило №5: новый цветовой токен определяем через палитру (`var(--base-XX)`), чтобы он сам работал в обеих темах. Литералы (`#fff`) — только если значение реально не зависит от темы.

---

## Шпаргалка «что брать»

- Фон страницы → `--background`
- Карточка/панель → `--surface` (или класс `.card`)
- Текст → `--text-primary` / `--text-secondary`
- Стилизуешь Material-компонент → роль `--mat-sys-*`
- Акцентная подсветка → пара `*-container` / `on-*-container`
- Отступ/радиус → токен `--padding-*` / `--radius-*`
- Шрифт → `appTypography` или миксин
